# 个人网站开发提示词文档

## 项目概述

基于 Astro + TailwindCSS 构建的个人网站，包含以下功能：
- 首页烟花特效 Banner
- 中英文语言切换
- 白天/夜间主题切换
- 博客搜索功能
- RSS 订阅
- 产品时间线展示

---

## 核心提示词模板

### 1. 功能开发类

#### Banner 烟花特效
```
创建一个具有烟花效果的 Hero Banner 组件：
- 夜空渐变背景
- 星星闪烁动画
- 点击或移动鼠标触发烟花
- 烟花颜色为冷白色
- 情侣剪影图片放在底部中央
- 鼠标移动到图片上不触发烟花效果
```

#### 语言切换功能
```
实现中英文切换功能：
- 导航栏显示语言切换按钮（EN/中）
- 切换时所有页面内容同步更新
- 使用 data-zh 和 data-en 属性存储双语文本
- 状态保存到 localStorage
- 设置 html lang 属性
```

#### 主题切换功能
```
实现白天/夜间模式切换：
- 使用滑动开关样式（Toggle Switch）
- 包含太阳/月亮图标
- 平滑的滑动动画效果
- 状态保存到 localStorage
- 支持系统偏好检测
```

### 2. 页面布局类

#### 时间线展示
```
产品页面使用时间线形式展示项目：
- 按年份分组
- 渐变色时间线连接线
- 每个项目卡片包含：月份、标题、描述、状态标签
- 支持中英文切换
- 悬停效果增强交互性
```

#### 博客搜索
```
博客页面添加搜索功能：
- 在 Header 组件中添加搜索框
- 实时过滤文章列表
- 支持标题和描述搜索
- 搜索框支持中英文 placeholder 切换
```

### 3. 样式调整类

#### 组件样式修改
```
修改 [组件名] 的样式：
- [具体位置] 改为 [新位置/大小]
- [颜色/背景] 改为 [新颜色]
- 添加 [动画/过渡效果]
- 移除 [不需要的元素]
```

## 最佳实践

### 代码组织
1. **组件分离**: 将可复用的 UI 元素抽取为独立组件
2. **语言支持**: 使用 `data-zh` 和 `data-en` 属性实现内容国际化
3. **样式管理**: 使用 TailwindCSS 工具类，避免自定义 CSS

### 提示词编写技巧

#### 好的提示词示例
```
✅ 将背景图片放在底部中央，移除提示文字，鼠标移动到图片上不触发烟花效果

✅ 烟花效果改为冷白色，不要彩色

✅ 时间线年份改为 2025，移除 2023 和 2022 的条目
```

#### 需要改进的提示词
```
❌ 改一下样式（太模糊，需要明确指出修改什么）

❌ 优化一下代码（需要明确优化目标）
```

### 提示词结构建议

1. **明确目标**: 说明要实现什么功能
2. **指定位置**: 明确修改哪个文件/组件
3. **描述细节**: 包含颜色、大小、位置等具体参数
4. **列出约束**: 说明不要什么、保留什么
